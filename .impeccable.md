# Impeccable — 프로젝트 디자인 컨텍스트

이 파일은 AI·협업 도구가 UI/카피 결정을 맞출 때 참고하는 **단일 출처**입니다. 코드와 불일치하면 코드·디자인 토큰을 우선하고, 이 문서를 갱신하세요.

## Design Context

### Users

- **주 사용자**: **교인 비중이 높음**. 동시에 **새가족·외부 방문자**도 분명히 고려한 사이트이며, **폐쇄적이지 않다** — 내비·카피·첫인상은 환영·열린 톤을 유지한다.
- **핵심 과업**: 예배/행사 정보 파악, 교회 소식·공지 확인, 연락처·위치 파악, 콘텐츠(뉴스·C-log) 읽기.
- **첫 방문자·온보딩(웹 맥락)**: 계정형 제품이 아니므로 **최소 성공**은 “**예배 시간·오시는 길(위치)·연락**을 스크롤 1~2번 안에 찾을 수 있는 것”으로 둔다. 메인 Quick Link·내비·푸터가 그 경로를 담당한다.
- **감정 목표**: 메타 카피의 **행복·꿈**과 브랜드 3단어(**행복, 활기, 신실함**)가 맞닿도록 — **따뜻함·에너지·믿음직함**을 동시에 느끼게 한다.

### Brand Personality

- **한 줄**: **현대적이고 열린** 교회 웹 — 딱딱한 전통 교회 사이트 틀을 벗어나되, 가벼운 유행만 쫓지 않는다.
- **3단어(공식)**: **행복 · 활기 · 신실함**
- **음성/톤**: 한국어 기본(`lang="ko"`), **명료·정중**하되 **환대**가 느껴지게. 사실·일정은 정확히, 문장은 차갑지 않게.

### Aesthetic Direction

- **베이스**: **라이트 모드**가 현재 중심(`--bg-page`, `--text-*`). **다크 모드는 도입 가능성을 열어 둔다** — 새 UI·토큰 설계 시 라이트/다크 확장을 막지 않는다.
- **색**: 공통 팔레트는 CSS 변수(`common/styles/constants/_colors.scss`). **액센트는 계절별 오버라이드**(`html[data-season]`). **본문·캡션 대비는 흐리멍덩하게 두지 않는다** — 특히 연령이 높은 교인을 고려해 **읽기 쉬운 명도**를 우선한다.
- **타이포**: 본문 **Pretendard Variable**, 포인트 **Useol**. **글자 크기 조절(텍스트만 키우는 옵션)** 은 바람직한 로드맵이나 아직 미구현 — 여유가 생기면 우선 검토할 기능으로 둔다.
- **레이아웃·반응형**: `vw()`·`font-vw()`·`mq()` 믹스인 기반. 메인은 둥근 하단·브랜드 보더 등 **세련된 랜딩** 인상.
- **모션**: Framer Motion, GSAP, Swiper는 **과하지 않게**. **화려한 모션은 일부 교인에게 부담**이 있었음 — 새 애니메이션은 **짧고 의미 있게**, `prefers-reduced-motion` 존중을 기본으로 한다.
- **레퍼런스**: **기업 홈페이지의 정돈·신뢰감** + **따뜻함** + **젊은 느낌의 비영리 재단** 사이트에 가까운 균형.
- **안티 레퍼런스**: **기존 교회 사이트 흔한 딱딱함·과한 보수적 톤만으로 묶인 UI**는 지양한다.

### Design Principles

1. **정보 우선**: 예배·공지·연락처 등 핵심 경로의 계층이 한눈에 보이게; 장식은 정보 뒤에 둔다.
2. **열린 교회**: 교인 중심이어도 **새가족·외부인이 소외되지 않는** 내비·카피·첫 화면을 유지한다.
3. **토큰 일관성**: 색은 `--text-*`, `--bg-*`, `--accent-*`와 계절 규칙을 따른다. 페이지 전용 색은 해당 페이지 최상위 SCSS 모듈에만 둔다(프로젝트 SCSS 규칙 준수).
4. **계절 리듬**: 액센트·섹션 이미지가 계절과 맞물리도록 유지한다. 서버 초기값과 클라이언트 테마(`ThemeManager`) 불일치를 피한다.
5. **타이포 하이라키**: Pretendard로 본문 가독성을 확보하고, Useol은 포인트에만 쓴다.
6. **접근성·포용**: WCAG를 **의식적으로 고려**(가능하면 **2.1 AA**를 지향). **고연령 교인**: 대비·크기·모션 부담을 실무에서 우선순위에 둔다. **다크 모드**는 향후 확장을 가정해 토큰·컴포넌트를 지나치게 라이트 전용으로 고정하지 않는다.
